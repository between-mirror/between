Get it

Two ways: build it from source now, or wait for the signed installer. The analysis is the same code either way. What you would pay for later is the installation — signing, guided setup, migrations, updates — not the program.

Build from source — available now

Free, and the same analysis the paid edition will run.

git clone https://github.com/between-mirror/between.git
cd between
npm ci                                   # exact versions from the lockfile
npm run demo:serve                       # try the fictional couple first

npx tsx server/src/cli/ingest.ts "data/sms-XXXX.xml"
npm run dev                              # → http://localhost:5273

Node 22 or newer. Node 20 is not enough on Windows: the native database module publishes no prebuilt binary for it there, so installing would need a full C++ toolchain. (Node 20 also reached end of life in April 2026.) This is the sort of thing that used to be wrong in our own documentation, so it is stated precisely.

Getting your messages off the phone

You produce the input yourself. Install SMS Backup & Restore (SyncTech) on the Android phone, back up to local storage — not to Google Drive or any cloud, then copy the resulting sms-….xml to your computer over a cable. Nothing passes through a service at any point.

A WhatsApp chat works the same way: export it from the app (one-to-one chats only) and hand over the .txt or the .zip. For anything else, the generic importer takes a CSV, JSON or JSONL file with a timestamp, a sender, a direction and a body. Imports merge rather than stack, so a newer backup adds what is new instead of duplicating what you already had.

Official Desktop — waitlist

A signed, one-click Windows installer: no Node, no Git, no terminal. Guided import, automatic migrations, signed updates, data management that asks about your archive separately from the app, and published build provenance for every release.

It is not built yet. The waitlist is a GitHub Discussion — react with 👍, and comment which platform you want. No email address is collected, because there is nowhere to put one and nothing that would send you mail.

Use the Discussion's own Subscribe button to hear when it exists. That is not a formality: since there is no mailing list, no account and nothing here that can contact you, GitHub's notification on that thread is the only way you will find out. A 👍 registers interest; it does not reach you later.

What "official" means

Only builds signed and published by the project's own organisation are Official Between Mirror builds. Anyone may fork the code and share their own — the licence allows it and so do we — but a build's origin is the one thing a person needs to be able to check when the software reads their private messages.
