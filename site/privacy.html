Privacy

There is no privacy policy in the usual sense, because there is no service. The project collects nothing, because there is nowhere for it to go — and the two places that is not the whole story, the host of this website and the update check the installer will one day carry, are spelled out below rather than left out.

What is collected

The project collects nothing. That is worth separating from what the host sees, because the two get blurred constantly and the blur always favours the person making the claim: this website is served by GitHub Pages, and GitHub receives the ordinary request metadata any web server receives — your IP address, which page you asked for, your browser's user-agent string. Between Mirror sets no cookies, ships no analytics, and loads no scripts, from here or anywhere. But a page cannot honestly tell you that nobody saw you ask for it.

What the application collects, to be specific, because "nothing" is a word every company uses:

  • No accounts, no sign-up, no email address, no identity.
  • No analytics, no usage statistics, no crash reporting, no session replay.
  • No licence check, no activation, no hardware fingerprint.
  • Nothing about you, your archive, or how you use the program — not in aggregate, not anonymised, not ever.

This is not an opt-out. There is no code that does these things, no dependency that could, and a test in the build that fails if one is added.

The one request the application will make on its own

Today it makes none. Today the source build makes no off-device request unless you explicitly choose an external model path. In local-only mode, analysis requests go only to the model running on your own machine. The signed installer that would carry an updater does not exist yet. What follows is the policy it will ship under, published now so it is a commitment made in the open rather than a decision taken later under pressure.

Checking whether a security update exists will be on by default, because a tool holding years of someone's messages should not sit quietly on a known vulnerability just to keep a slogan tidy — and there will be a one-click switch to turn it off.

It will carry no identifier: one static release-manifest file, the same file for everybody, with no install ID, no machine ID, no version history, no counter, and nothing about your archive. Whoever serves that file learns what any file host learns — that some computer asked for a public file. That is the whole of it.

Telemetry remains permanently absent, and an update check is not telemetry: one is the program asking a question, the other is the program reporting on you. If that distinction ever stops holding here, it is a broken promise and should be called one.

Where your archive lives

In one SQLite file on your computer, next to the application. The server binds to loopback only and refuses to start otherwise. There is no application code that collects or reports telemetry, no dependency included for analytics or monitoring, and the build fails if a known telemetry dependency or collection path is introduced.

The application also has a Your data panel that tells you the database's exact path and size, what you imported it from, and where exports go — with buttons to check it is intact, copy it somewhere safe, and delete all of it. Leaving has to be one of the things you can do, or "your data stays with you" is only half a promise.

The three paths any text can take

The counting — river, episodes, eras, patterns, exports — never leaves your machine. Only the written readings need a language model, and there are exactly three ways to run one. You choose, in Settings, and the default is the first:

Things we will never build

Permanent, and written down in the repository's roadmap so that a future version breaking one is visible as a broken promise rather than a product decision: no hosted service, no accounts, no telemetry, no DRM or licence servers, no per-reading fees, no "premium insight", no evidence-grade or court-ready claims, no selling or training on anyone's messages, no social features, no always-on agent watching new messages arrive.

What this does not protect you from

Local-first is not the same as safe. Anyone with your unlocked computer has your archive; a cloud-synced folder will copy it off the machine (the app warns at boot if it looks like one); the database is not encrypted at rest, deliberately, because a forgotten passphrase on a tool with no accounts means losing something irreplaceable. All of it is written out in the threat model rather than left for you to discover.
